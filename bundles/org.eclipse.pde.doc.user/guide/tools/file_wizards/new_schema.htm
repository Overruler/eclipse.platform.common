Extension Point Schema

A target definition specifies all aspects of a target, including its location, constituent plug-ins and environment.

The New Extension Point Schema wizard is available under File > New > Other... > Plug-in Development > Extension Point Schema from the top level menu.

 

Extension Point

When you create the schema, you need to specify the name and fully qualified ID (i.e. the concatenation of the plug-in ID and extension point ID) of the extension point that the schema is associated with.

 

Schema

The Container field refers to an existing location in the workspace where the schema will be created.

The Extension Point Schema field refers the path to the schema within the specified container. The schema must have a .exsd file extension.

Some schemas (e.g. schema for an expression language) may be incorporated in other schemas. These shared schemas are marked as such when the Create shared schema for inclusion option is selected.

When checked, the Edit extension point schema when done option opens the newly created .exsd file in a specialized PDE schema editor.
