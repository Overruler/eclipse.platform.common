Settings Tab

The Setting Tab allows you to customize the Java runtime environment and configuration area settings of the launch configuration.

 

Java Executable

The Java Executable option allows you to specify the executable to run the Equinox OSGi framwork with. On Windows, for example, the default executable is javaw.exe, which you can override and use java.exe, if desired.

 

Java Runtime Environment

You can test your bundle against the Java Runtime Environment (JRE) of your choice, and this runtime JRE does not necessarily have to be the same JRE against which the bundle compiles in the workspace.

The Runtime JRE combo box is populated with all the JREs listed on the Windows > Preferences.... > Java > Installed JREs preference page. You can dynamically add more JREs to the list of installed JREs by pressing the Installed JREs... button, which will open the Installed JREs preference page.

 

Bootstrap Entries

By default, the only entry on the launch configuration classpath is the ${target_home}/startup.jar, which houses the org.eclipse.core.launcher.Main class that contains the bootstrap code. ${target_home} refers to the location of the target platform as specified on the Window > Preferences... > Plug-in Development > Target Platform preference page.

The Bootstrap Entries field allows you to specify and append additional locations to the classpath, if required.

 

Configuration Area

Every Equinox OSGi framework instance has a unique configuration area which contains metadata, caches and configuration files read by the framework and possibly other bundles.

Typically, the location of the configuration is not important and the user needs not to worry about its contents. PDE therefore generates a unique configuration area for every launch in its own metadata area. This configuration area is deleted when the launch configuration is deleted.

The location of the configuration area can be set by the user, if desired. Variables in the directory path are supported, thus making launch configurations sharable among team members.

You also have the option to clear the configuration area prior to launching, which you may find useful if you are testing how bundles cache data in that area.
