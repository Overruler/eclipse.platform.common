Overview

The Overview page allows you to view and edit the name and locations of your target definition. As well, the Overview page provides hot links to Content and Environment information.

 

Target Definition

The Target Name is a translatable string that identifies your target definition in the Target Selection combo box on the Target Platform preference page.

The Target Location specifies the location to which the target platform location should be set. For convenience, the editor provides the option to specify that The target platform is the same as the host (running) platform. This option should be used if you wish to develop plug-ins for your current host environment. Users can also specify an absolute Location other than the host. To aid in portability of target definitions, the path utilizes Debug String Substitution variables.

 

Additional Locations

Target definitions allow you to include required plug-ins from Additional Locations other than the main target location. Plug-ins found in these locations will be included in the target platform. These locations need to be absolute, but can consist of Debug String Substitution variables to enhance portability.
