Included Plug-ins

The Plug-ins page lists all the plug-ins and fragments that should be packaged in the feature and specifies their properties

 

Plug-in List

The plug-in list enumerates all the plug-ins and fragments that are included directly in the feature.

Press the Add... button to show the list of available plug-ins and add plug-ins to the list.

The preferred versioning scheme is for the plug-ins referenced to be given a 0.0.0 version. This is a special value that will be replaced by the actual plug-in version when exporting the feature. This option is especially convenient when plug-in versions contain a qualifier segment.

 

Plug-in Details

Plug-in details are used as guidelines by Update Manager to determine what, when and how to install the plug-ins included by a feature.

The download size and installation size are optional hints that can be provided to allow Update Manager to provide useful feedback to the user about the feature being installed.

The unpack option determines whether a plug-in is to remain intact in a JAR after installation. The default is to unpack (i.e. flatten).

If a plug-in or fragment contains platform-specific code, then environment conditions should be specified and associated with that plug-in or fragment.
