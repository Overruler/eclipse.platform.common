Overview

The Overview page serves a dual purpose:

  1. It contains two main sections, General Information and Supported Environments, that define important properties of the feature..
  2. It functions as a quick reference on how to define, export and publish a feature by providing the Feature Content, Exporting and Publishing sections. These sections provides hyperlinks which you can follow to navigate to pages or invoke commands.

 

General Information

An ID is mandatory and must comply with the specifications.

A version is mandatory and must be of the form major.minor.micro.qualifier (e.g. 1.3.0).

A name is the translatable presentation name of the feature. This field is required.

A provider is the translatable name of the feature provider. This field is optional.

Branding information for primary features is stored in a branding plug-in. If not explicitly set, Eclipse will assume that the branding plug-in has the same identifier as the feature.

The update site URL denotes a URL that is used by Update Manager when searching for updates to your feature.

The update site name is the human-readable label of the referenced update site.

 

Supported Environments

By default, your feature is treated as universally portable. You can add constraints by providing supported operating/windowing systems, languages and/or system architectures. This information will be used to ensure that your feature is not installed or shown in the context that does not match these constraints.
