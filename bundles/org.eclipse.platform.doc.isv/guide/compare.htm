Compare support

The Compare plugin (named org.eclipse.compare) implements:

  • Two and Three-way compare/merge components for hierarchical structures superimposed on text
  • Differencing engines for:
    • hierarchical structures
    • character ranges, e.g. text lines or words
  • Integration of patch files
  • Comparison and merging with the local history.
In order to be extensible to different content types the infrastructure defines the following extension points:

  • org.eclipse.compare.contentMergeViewers
    A content merge viewer performs a two-way or three-way compare of its inputs, presents the result side-by-side or in any other suitable way, and lets the user merge between the inputs. Typical examples for content merge viewers are compare viewers for text or images.
  • org.eclipse.compare.structureMergeViewers
    A structure merge viewer performs a two-way or three-way compare of its inputs, typically presents the result in a hierarchical way, and lets the user merge between the inputs. Typical examples for structure merge viewers are viewers showing workspace resources or the members of an archive file.

Because the implementation of most structure compare viewers is based on a tree and therefore very similar, the compare plugin provides a generic tree-based StructureDiffViewer that defines its own extension point org.eclipse.compare.structureCreators.

A structureCreator breaks a single input object into a hierarchical structure. The StructureDiffViewer performs the compare on the resulting structure and displays the result as a tree.

In addition there is an extension point org.eclipse.compare.contentViewers for content viewers that do not compare their inputs. A contentViewer is used in places where only a single input is available and therefore no compare is necessary. A typical example for this is the "Restore from Local History" function.

All compare viewers are standard JFace viewers that expect an input object implementing the ICompareInput interface.

For implementing viewers the compare plugin provides additional support for:

  • content oriented compare and merge viewers
  • structure oriented compare and merge viewers.
